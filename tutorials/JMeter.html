    # 章节5基础知识--12类测试元件及其作用域、执行顺序

    # [5.1]--课时13十二类元件概述02:16

    • 十二类元件概述
      1. 测试计划 测试脚本的容器，其他元件都会包含在它当中
      2. 线程组 测试计划执行的入口，线程组的每个线程都独立运行测试脚本，一个线程相当于一个用户，通过多线程并发执行实现并发性能测试
      3. 逻辑控制器 指定它下面的取样器是如何发送请求以及这些取样器的执行顺序
      4. 取样器 完成JMeter实际工作，每一个取样器生成一个或多个取样结果
      5. 配置元件 可以修改请求
      6. 前置处理器 在取样器运行之前执行，主要目的是修改取样器或更新变量
      7. 后置处理器 在取样器运行之后执行，主要用于处理响应数据
      8. 定时器 设置请求之间的时间间隔
      9. 断言 对响应结果进行验证
      10. 监听器 可以监听、查看、保存测试结果
      11. 测试片段 辅助组件，它不会被执行
      12. 非测试元件 主要是完成脚本的录制或者辅助调试

    # [5.2]--课时14测试计划03:07

    测试计划 测试脚本的容器，其他元件都会包含在它当中

    1. 定义用户变量，是全局变量
      • 引用变量的方法：${变量名}
    2. Run Thread Groups consecutively(i.e. one at a time)
      • 独立运行每个线程组 【默认不勾选，标识各线程组间并行执行，勾选，则标识串行执行】
    3. Run tearDown Thread Groups after shutdown of main threads
      • 勾选 主线程结束后执行tearDown线程组；如不勾选tearDown线程组是不会被执行的
    4. Functional TestMode(i.e. save Response Data and Sampler Data)
      • 函数测试模式 【功能测试模式：取样器和响应结果会被保存，该模式可能对性能测试有影响】
    5. 将目录或者jar添加到classpath

    # [5.3]--课时15线程组09:39

    线程组 测试计划执行的入口，线程组的每个线程都独立运行测试脚本，一个线程相当于一个用户，通过多线程并发执行实现并发性能测试

    • 指定取样器出错后，要采取的动作：
      1. 继续
      2. 启动下一个线程循环 本次循环结束，执行下一个线程循环
      3. 停止线程 虚拟用户执行出错之后，后边就不再执行了，也不再循环了
      4. 停止测试 所有正在执行的取样器，执行结束后，程序停止
      5. 立刻停止测试 所有的线程马上停止
